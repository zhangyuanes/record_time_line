# 01 需求规格说明

## 1. 目标
构建一个本地优先、项目制的时间线创作平台，支持高质量前端编辑与预览体验，并可将完整时间线导出为可分享的离线网页包。

## 2. 范围界定
- [必须] 支持项目化管理：可创建、打开、重命名、归档本地项目。
- [必须] 支持时间线节点编辑：选择日期时间、编辑文本、添加图片/视频/链接、保存节点。
- [必须] 支持实时预览：编辑态与预览态同步，支持拖动时间线、查看单节点、过渡动效。
- [必须] 支持主题系统：至少支持 3 种主题布局与动效策略，并允许后续扩展。
- [必须] 支持导出分享：导出后包含 `index.html`、数据文件、媒体资源，接收方解压后直接打开可查看完整时间线。
- [必须] 支持纯本地存储：数据与媒体仅保存在本地文件夹，不依赖云端服务。
- [必须] 前端体验优先：重点保障界面美观、动效自然、交互反馈明确、长时间线下的性能稳定。
- [增强] 可选资源压缩（图片压缩、视频缩略图）以降低导出包体积。
- [增强] 可选背景音乐、封面视觉、高光节点等叙事增强能力。
- [排除] 多人协作、在线账户体系、云端同步。
- [排除] 移动端原生 App（本期以 Web 本地平台为主）。
- [排除] AI 自动生成内容（本期仅提供人工创作流程）。

## 3. 用户故事/功能流程
1. 用户打开平台，进入项目列表并创建新项目。
2. 用户进入编辑工作台，添加时间节点（日期时间 + 文本 + 媒体/链接）。
3. 用户在右侧或独立区域实时预览时间线效果，切换主题并调整节点顺序与展示重点。
4. 用户保存项目，系统将数据与媒体写入项目目录。
5. 用户触发导出，系统生成离线可运行的分享包。
6. 朋友解压后打开 `index.html`，在浏览器中完整查看该时间线内容与动效。

## 4. 验收标准（AC）
- [ ] AC-01 项目管理：可在本地创建并维护多个项目，项目间数据隔离。
- [ ] AC-02 节点编辑：节点支持时间、标题、正文、媒体、链接字段，保存后刷新仍可还原。
- [ ] AC-03 实时预览：编辑后 1 秒内可见预览变化，且不出现明显闪烁和布局跳变。
- [ ] AC-04 主题切换：至少 3 套可用主题，切换后布局与动效表现明显不同且内容一致。
- [ ] AC-05 导出可用：导出目录包含完整数据与资源，第三方机器无需安装 Node.js 即可通过浏览器打开 `index.html` 查看。
- [ ] AC-06 本地优先：核心功能在离线环境可用（新建、编辑、预览、导出）。
- [ ] AC-07 性能门槛：包含 200 个节点、每节点 1 张图时，编辑与滚动仍保持可接受交互流畅度。
- [ ] AC-08 前端质量：关键交互（新增节点、拖拽定位、主题切换、查看单节点）具备明确状态反馈与一致视觉规范。

## 5. 前端高优先级设计输入约束
- 用户后续会提供原型图与动效要求，本项目实现必须以该输入为一等约束。
- 若原型与当前规格冲突，以原型中的交互意图优先，并在 `03_implementation.md` 的执行步骤中记录变更点。
- 动效策略应可配置（时长、缓动、触发条件），避免硬编码在单一组件中。

## 6. 风险与待确认
- 风险-01 本地浏览器对媒体编解码支持差异可能影响导出体验。
- 风险-02 大体积媒体在低性能机器上可能导致首次打开较慢。
- 风险-03 前端高保真动效若无分级降级策略，可能在长时间线场景产生卡顿。
